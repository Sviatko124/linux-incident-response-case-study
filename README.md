# Linux Incident Response Case Study

A portfolio project demonstrating the complete lifecycle of a simulated Linux security incident, from initial compromise to incident response, threat hunting, and detection engineering.

## Overview

This repository documents a realistic attack against a Linux host and the subsequent investigation from the defender's perspective.

Unlike many offensive security labs that stop after obtaining shell access, this project follows the incident through every major stage of the response process:

- Initial compromise
- Evidence collection
- Timeline reconstruction
- Detection and triage
- Containment
- Eradication
- Recovery
- Threat hunting
- IOC collection
- MITRE ATT&CK mapping
- Network analysis
- Detection engineering

The project emphasizes documentation, repeatability, and evidence-based investigation rather than exploitation alone.

## Project Goals

- Simulate a realistic Linux compromise.
- Reconstruct the incident using host and network evidence.
- Produce documentation similar to a professional incident response engagement.
- Develop reusable detection content based on observed attacker behavior.
